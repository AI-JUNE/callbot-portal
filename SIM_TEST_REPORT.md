# CallBot 무과금 시나리오 검증 리포트

- 일자: 2026-07-07
- 대상: https://callbot-portal.vercel.app · `/api/sim_call`
- 빌드: B76 (검증 시점 라이브 B75 → 폴리시 반영 후 B76 배포 대기)
- 과금: 전 항목 **0원(텔레포니 미발생)** · LLM(gemini-2.5-flash) 토큰만 극소액

## 배포/엔진 조치 요약
- 시나리오 2개 → **8개**로 확장 (`api/sim_call.py`)
- CPaaS 화면에 시나리오 버튼 8개 배치 (`index.html`, `public/index.html`)
- `engine.py` `maxOutputTokens` 256 → **1024**: 복잡한 턴에서 응답이 중간에 잘리던 문제 해결
- integrity 프롬프트 수정: 모르는 지점명·날짜를 `[날짜]`처럼 대괄호로 노출하지 않도록 정정

## 시나리오별 결과

| 시나리오 | 경로 | 결과 | 비고 |
|---|---|---|---|
| refund | 환불 접수 | 정상 | 초기 검증 완료 |
| order | 주문/배송 조회 | 정상 | 3턴 완결, 실제 주문가 인용 |
| redelivery | 재배달 | 주의 | 아래 참고 |
| handoff | 상담사 연결 | 정상 | `transferred: true` 발동 확인 |
| refund_over | 과다환불 방어 | 정상 | 실제가 159,000원 인용·과다금액 거절 |
| care | 아웃바운드 안부 | 정상 | 의료조언 회피 확인 |
| integrity | 청렴도 조사 | 정상 | 8턴 완결(자리표시자 노출은 폴리시로 수정) |
| overdue | 연체 안내 | 정상 | 의도분류(모름/상환예정/종료) 정확 |

## 남은 개선점 (선택)
1. **redelivery 재배달 접수 경로 미검증**: 목업 주문 데이터상 해당 상품이 "배송완료"라
   봇이 재배달을 정책상 거절함(안전 동작). 실제 `request_redelivery` 접수 흐름을 테스트하려면
   `engine.py`의 목업 주문(`_ORDER`)에 "미배송/누락" 상품을 하나 추가해야 함.
   → 이 데이터는 refund/order 시나리오와 공유되므로, 변경 전 확인 필요.
2. handoff 전환 태그가 2번째 턴에서 확정됨(엔진이 모델의 escalate 호출을 같은 턴에 반영하지 않음).
   기능상 문제는 없으나, 원하면 같은 턴에 즉시 반영되도록 소폭 개선 가능.

## 안전
- CPAAS_LIVE는 계속 **꺼짐** 상태 유지 → 실통화/과금 없음
- 실제 전화 연동(라이브)은 준비 완료 후 사용자가 직접 켤 때만 과금
